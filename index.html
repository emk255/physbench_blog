PhysBench

Elliot Kim
Cornell University

Introduction

Prior research has shown promising gains when augmenting agents with external tools such as web search [1]. However, there is little systematic research that isolates the effect of built-in web search tools across domains. This report presents a surprising case study: on challenging physics questions, models equipped with built-in web search perform worse than the same models without it. We therefore introduce PhysBench, a contamination-free benchmark of 843 curated questions spanning mechanics, electromagnetism, thermodynamics, optics, and relativity, including 696 multimodal problems with images. PhysBench is substantially more challenging than existing benchmarks and demonstrates that augmenting a built-in web search tool is adversarial to models' performance.

Key Contributions

Challenging Physics Benchmark for Multi-Modal Models

Contamination-free; tests conceptual reasoning over computational ability.

Adversarial Effect of Built-in Web Search Tools

Isolates the adversarial effect of state-of-the-art built-in web search tools on multi-modal models.

Experiments

Setup

Submatrix Analysis

To quantify the relative difficulty of our benchmark $B_{\text{Phys}}$ in comparison to existing benchmarks $B_{\text{Existing}}$, we first classify a group of strong models $M_{\text{strong}}$ and weak models $M_{\text{weak}}$ using results from Table 1 with an accuracy rate threshold of $t=0.30$. We then identify sub-matrices of the performance dataset where:

  1. (1) $A(M_{\text{Strong}}, B_{\text{Phys}}) \gg A(M_{\text{Weak}}, B_{\text{Phys}})$, and
  2. (2) $A(M_{\text{Strong}}, B_{\text{Existing}}) \approx A(M_{\text{Weak}}, B_{\text{Existing}})$,

where $A(M,B)$ refers to the average accuracy rate of models in group $M$ on benchmark $B$. In the following group of experiments, $B_{\text{Existing}}=\text{OlympiadBench}$, as it represents the most comparable existing multi-modal physics benchmark in terms of domain coverage and problem difficulty.

Clustering Analysis

We extend submatrix analysis through clustering to discover natural problem groupings beyond predefined features. We generate embeddings directly from problem text (context + question) for all problems across both benchmarks, placing them in a unified vector space. After clustering, we use an LLM to generate keyword lists for problems within each cluster, enabling qualitative interpretation of the discovered subcategories.

Results

PhysBench exhibits substantial model discrimination even on problems with simple math, revealing that difficulty arises from conceptual reasoning rather than computation. Clustering analysis further identifies thermodynamics as particularly challenging, with persistent performance gaps on real-system physics like phase transitions.

Case Study: Augmenting Built-in Web Search Tool

With state-of-the-art LLMs now offering built-in web search tools that combine conversational and information retrieval capabilities, we hypothesize these integrated architectures will improve overall performance.

Note: For OpenAI models, we "require" web search to be used because "auto" mode never utilized web search. Google's models does not have such option, as there is only "auto" mode.

Results

Web search tools degrade rather than improve performance on PhysBench, with larger negative effects on weaker models—confirming the benchmark tests conceptual inference over recall, where external evidence distracts more than it helps.
