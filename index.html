Web Search is Surprisingly Adversarial to Physics Reasoning

Elliot Kim
Cornell University

Introduction

Prior research has shown promising gains when augmenting agents with external tools such as web search [1]. However, there is little systematic research that isolates the effect of built-in web search tools across domains. This report presents a surprising case study: on challenging physics questions, models equipped with built-in web search perform worse than the same models without it. We evaluate state-of-the-art multimodal models on a curated set of 843 contamination-free physics problems spanning mechanics, electromagnetism, thermodynamics, optics, and relativity, including 696 multimodal problems with images. Our analysis reveals that web search degrades performance across all models, with larger negative effects on weaker models—demonstrating a domain where additional information retrieval actively harms reasoning rather than helping it.

Experimental Results

With state-of-the-art LLMs now offering built-in web search tools that combine conversational and information retrieval capabilities, we hypothesize these integrated architectures will improve overall performance. Web search tools degrade rather than improve performance on a challenging set of physics problems, with larger negative effects on weaker models.

Note: For OpenAI models, we "require" web search to be used because "auto" mode never utilized web search. Google's models does not have such option, as there is only "auto" mode.

Case Study

To understand how web search degrades model performance on physics reasoning, we examine specific cases where Gpt-4.1-mini with web search tools produce incorrect solutions that differ from their toolless counterparts. Two representative examples illustrate different failure modes.

Case 1: Sources with OCR Issues

Question 260: A physics problem requiring calculation of classical specific heat at constant volume per electron.

✓ No Tools (Correct):

"1. Classically, electrons behave like an ideal gas with 3 degrees of freedom, each contributing (1/2)k_B per electron to the internal energy. Therefore, the classical specific heat at constant volume per electron is c_v = (3/2) k_B."

✗ With Web Search (Incorrect):

"In classical theory, the specific heat at constant volume per electron is calculated using the equipartition theorem, which states that each degree of freedom contributes k_B to the average energy. For a free electron gas, each electron has three translational degrees of freedom, leading to an average energy of 3k_B T per electron. Therefore, the classical specific heat per electron is c_v = 3k_B."

Root Cause: The model referenced an accurate PDF source that correctly stated "1/2 k_B to the heat capacity." However, the associated plain text extraction of the same source (left panel) incorrectly rendered this as "kB to the heat capacity." The model picked up the incorrectly translated text rather than the correct PDF content, essentially leading to use the wrong formula.

Case 2: Sources with Incorrect Formulas

Question 272: A relativistic elastic collision problem requiring center-of-mass frame calculations.

✓ No Tools (Correct):

"Find velocity of the center-of-mass (COM) frame using total momentum and total energy: β = (p1 + p2) / (E1 + E2) = (1.5 + 0.4364) / (2.5 + 1.091) = 1.9364 / 3.591 = 0.5393."

✗ With Web Search (Incorrect):

"The velocity of the COM frame, denoted as β, is given by the formula: β = (m1 * v1 + m2 * v2) / (m1 + m2) ... Substituting the given values: β = (2.0 kg * 0.6c + 1.0 kg * 0.4c) / (2.0 kg + 1.0 kg) ... β = (1.2c + 0.4c) / 3.0 kg ... β = 1.6c / 3.0 kg ... β = 0.5333"

Root Cause: Of the 12 sources referenced, 50% were classical mechanics sources that provided inappropriate formulas for a relativistic problem. The model selected contradictory information and applied classical physics to a scenario requiring relativistic treatment, resulting in fundamentally incorrect reasoning.

Analysis

These case studies reveal two critical failure modes when models use web search for physics reasoning:

  1. Information Extraction Errors: Even when sources are accurate, text extraction errors can introduce subtle but significant mistakes that models incorporate into their reasoning.
  2. Inappropriate Source Selection: Web search can retrieve contradictory or irrelevant information that leads models away from correct physical principles.

Both cases confirm the existence of the web search's adversarial effect phenomenon. Web search introduces noise into the reasoning process, aligning with previous phenomenon like "Lost in the Middle", where models struggled to distinguish valuable/accurate information when flooded with large amounts of data. The case studies call for potential methodologies to effectively utilize domain filtering.
